# Flask 실습 및 네트워크 기초 정리

## Run Without Debugging vs Start Debugging

| 구분 | Run Without Debugging | Start Debugging |
|------|------------------------|-----------------|
| 실행 방식 | 단순 실행 | 디버거 실행 |
| 속도 | 빠름 | 느림 |
| 중단점 | 사용 불가 | 사용 가능 |
| 변수/스택 확인 | 불가 | 가능 |
| 예외 발생 시 | 프로그램 종료 | 멈추고 분석 가능 |
| 용도 | 결과 빠른 확인 | 흐름/버그 추적 |

요약:
- 결과 확인: Run Without Debugging
- 코드 분석/디버깅: Start Debugging

## Flask의 역할

- Python 기반 경량 웹 프레임워크
- HTTP 요청을 처리하고 응답 반환
- 주요 기능:
  - URL 라우팅 (@app.route)
  - 요청/응답 처리
  - 템플릿 렌더링 (Jinja2)
  - 개발용 서버 제공
- 가볍고 빠르게 웹 애플리케이션 개발 가능

한줄 요약: Flask는 Python으로 웹 애플리케이션을 쉽게 구축하는 경량 웹 프레임워크이다.

## 0.0.0.0의 의미와 장단점

의미:
- 모든 IPv4 네트워크 인터페이스에서 연결 수락
- 동일 네트워크(와이파이, 유선망) 다른 기기에서 접속 가능

| 장점 | 단점 |
|------|------|
| 다른 기기 접속 가능 (내부망 테스트 가능) | 외부 접속 가능으로 인한 보안 위험 |
| 팀 프로젝트 개발 시 편리 | 노출 주의 필요 |

실습 팁:
- 로컬 확인: 127.0.0.1
- 다른 기기 테스트: 0.0.0.0 설정

## 127.0.0.1 vs 내부 IP 접속

- 127.0.0.1 (localhost):
  - 자기 컴퓨터에서만 접속 가능
  - 외부 접속 불가

- 내부 IP (예: 192.168.x.x):
  - 동일 네트워크의 다른 기기에서도 접속 가능
  - 스마트폰, 다른 PC 접속 테스트 가능

요약: 로컬 테스트는 127.0.0.1, 다른 기기 접속 확인은 내부 IP 사용.

## 포트 번호의 의미와 충돌 해결

### 의미

- 한 컴퓨터 내 여러 프로그램의 네트워크 통신 구분 번호
- 예:
  - HTTP: 80
  - HTTPS: 443
  - Flask 기본: 5000

### 충돌 시 해결

1. 다른 포트 사용:
```python
app.run(port=5001)
```

2. 충돌 프로세스 종료:
- macOS/Linux:
```bash
lsof -i :5000
kill -9 [PID]
```

- Windows:
```powershell
netstat -ano | findstr :5000
taskkill /PID [PID] /F
```

3. PC 재부팅 (빠른 임시 해결)
